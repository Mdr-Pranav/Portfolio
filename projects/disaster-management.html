National Disaster Management

A website meant to keep track of vital resources/operations to help improve efficiency of managing Disasters and Volunteers

HTML JavaScript Node.js MySQL Sequelize
Repository

Project Overview

The National Disaster Management Platform is a comprehensive web-based system designed to facilitate coordination between NGOs, volunteers, and administrators during disaster relief operations. The system centralizes critical information about ongoing disasters, available resources, and volunteer deployments, significantly improving response efficiency and resource allocation.

Key Features

  • User Registration and Authentication

    Secure registration and login system with role-based access control for administrators, NGOs, and volunteers. Each user type has a customized dashboard with relevant functions and permissions.

  • Admin Management Tools

    Administrative interface for creating and managing training sessions, approving volunteer applications, and overseeing disaster relief operations. Includes real-time statistics and reporting capabilities.

  • Volunteer Management System

    Volunteers can browse current disasters, submit applications to assist, register for training sessions, and track their participation history. The system includes skill matching to connect volunteers with appropriate opportunities.

  • Disaster Tracking

    Comprehensive database of ongoing and past disasters with detailed information, resource requirements, and current status. Includes geolocation data and severity classification.

  • Database Integrity

    Implemented cascading database operations to maintain data consistency across related tables, ensuring that all records are properly updated or removed when parent records change.

Technical Implementation

Frontend

The user interface was built using HTML, CSS, and JavaScript to create a responsive design that works across devices. Key components include:

  • Interactive dashboards with real-time updates
  • Form validation for user inputs
  • Data visualization for disaster statistics
  • Responsive layouts for mobile and desktop access

Backend

The server-side implementation leverages Node.js with the following technologies:

  • Node.js: Server runtime environment
  • Express.js: Web application framework
  • MySQL: Relational database for storing user, disaster, and training data
  • Sequelize ORM: Object-Relational Mapping for database interactions
  • JWT Authentication: Secure token-based user authentication

Database Design

The relational database structure includes the following main tables:

  • Users: Stores user credentials and profile information with role designations
  • Disasters: Contains details about each disaster including location, type, severity, and status
  • Volunteers: Tracks volunteer skills, availability, and verification status
  • TrainingSessions: Manages upcoming and past training opportunities
  • DisasterApplications: Records volunteer applications for specific disasters
  • TrainingRegistrations: Connects volunteers with training sessions they've registered for

Setup and Installation

The application requires the following prerequisites:

  • Node.js and npm
  • MySQL Server
  • Git (optional)
  • Sequelize CLI (optional)

Installation involves:

  1. Cloning the repository
  2. Installing dependencies with npm
  3. Configuring environment variables
  4. Setting up the database
  5. Running the server

Challenges & Solutions

Challenge

Designing a database schema that efficiently handles the complex relationships between users, disasters, volunteers, and training sessions while maintaining data integrity.

Solution

Implemented a normalized database design with foreign key constraints and cascading operations. Used Sequelize ORM to manage model associations and transactions, ensuring consistent database states even during complex operations.

Challenge

Creating a secure yet user-friendly authorization system that enforces proper access controls while remaining accessible to users with varying technical skills.

Solution

Developed a role-based access control system using JWT tokens with appropriate expiration policies. Created middleware to validate permissions for each route, and implemented intuitive UI elements that only display actions the current user is authorized to perform.

Learning Outcomes

This project provided valuable experience in:

  • Designing and implementing complex database schemas with relationships
  • Creating role-based access control systems
  • Building a full-stack web application with Node.js and MySQL
  • Managing data integrity in a multi-user environment
  • Implementing user authentication and session management
